问题场景 ​

用了几年IDEA，发现很多同事还在用最基础的功能——打断点、F8单步、看变量值。IDEA的调试和效率功能远不止这些。这篇文章把我日常用得最多的技巧整理出来，都是在真实开发场景中验证过的。

远程调试：在线上环境Debug ​

线上出了问题，日志看不出原因时，远程调试是最高效的排查手段（但只能在测试环境用，生产环境慎用）。

Step 1：启动应用时开启远程调试

bash
# Java 9+ 使用的方式
java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005 -jar app.jar

# Java 8 的方式（旧写法）
java -Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=5005 -jar app.jar

关键参数：

  • suspend=n：不暂停启动，等调试器连接后再继续。如果设为 y，应用会卡在启动等你连
  • address=5005：监听端口，确保防火墙允许

Docker容器中开启远程调试：

dockerfile
ENV JAVA_TOOL_OPTIONS="-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005"
EXPOSE 5005

Step 2：IDEA配置远程连接

Run -> Edit Configurations -> + -> Remote JVM Debug：

  • Host: 测试服务器IP
  • Port: 5005
  • Use module classpath: 选择你的项目模块

连接成功后，跟本地调试一样打断点。

踩坑点：远程调试时本地代码要和远程部署的代码版本一致，否则断点位置会错乱。打完断点记得及时移除，否则会影响线上性能。

条件断点与高级断点 ​

条件断点

不是所有断点都要在每次命中时暂停。右键断点，输入条件表达式：

java
// 只在订单金额大于1万时暂停
order.getAmount() > 10000

// 只在特定用户ID时暂停
"user_12345".equals(userId)

// 只在第N次命中时暂停（Hit Count）
// 在断点属性中设置 Hit Count = 100，第100次命中时暂停

日志断点（不暂停的断点）

有时候你不想暂停程序执行，只想看某个变量的值。右键断点：

  1. 取消 "Suspend" 勾选
  2. 勾选 "Evaluate and log"
  3. 输入表达式："当前用户: " + user.getId() + ", 订单数: " + orders.size()

这样值会输出到IDEA的Console窗口，程序不会暂停。

方法断点

在接口方法名左边的槽位点击，会出现一个菱形断点。当实现类被调用时会暂停。适合调试Spring的AOP和代理对象。

字段断点（Field Watchpoint）

在类的字段声明处打断点，当这个字段被读取或修改时暂停。适合排查"这个值什么时候被改了"的问题。

热部署 ​

Spring Boot DevTools

最简单的热部署方案，修改代码后自动重启：

xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <optional>true</optional>
</dependency>

在IDEA中还需要开启自动编译：

  1. Settings -> Build -> Compiler -> Build project automatically 勾选
  2. Settings -> Advanced Settings -> Allow auto-make to start even if developed application is currently running 勾选

JRebel（付费方案）

真正的热加载，不需要重启。修改方法体后实时生效，速度比DevTools快得多。团队有预算的话推荐购买。

DCEVM + HotSwap Agent（免费方案）

开源的热加载方案，支持添加新方法、新字段（标准JVM HotSwap只支持修改方法体）：

bash
# 安装DCEVM（需要替换JVM的hotswap目录）
# 下载地址: https://dcevm.github.io/
java -XXaltjvm=dcevm -jar app.jar

常用插件推荐 ​

必备插件：

  1. Lombok：使用Lombok注解必须装，否则IDEA到处报红
  2. MyBatisX：在Mapper接口和XML之间快速跳转，配合MyBatis开发必备
  3. GenerateAllSetter：光标放在对象变量上，自动生成所有setter调用。从DTO转VO时特别省时间
  4. Translation：直接在IDEA里翻译文档注释，不用切浏览器
  5. Grep Console：Console日志按级别着色，ERROR红色、WARN黄色，一眼就能看到报错

提升效率的插件：

  1. Key Promoter X：你用鼠标点了菜单操作时，会提示对应的快捷键，逼你养成快捷键习惯
  2. String Manipulation：字符串格式转换（驼峰、下划线、大写、小写），处理JSON字段名时很方便
  3. RestfulTool：自动扫描项目中所有Controller的接口，快速导航和测试

代码模板（Live Templates） ​

IDEA自带的代码模板很好用，但自定义模板更强大。

路径：Settings -> Editor -> Live Templates

自定义模板示例：

1. 日志模板

缩写: logi
模板: log.info("【$CLASS_NAME$】$METHOD_NAME$: $END$");

2. try-catch模板

缩写: tryc
模板:
try {
    $SELECTION$
} catch (Exception e) {
    log.error("$CLASS_NAME$.$METHOD_NAME$异常", e);
    throw new BizException("操作失败");
}

3. 单元测试模板

缩写: test
模板:
@Test
public void $METHOD_NAME$() {
    // given
    $END$
    
    // when
    
    // then
}

4. Spring Controller模板

缩写: apim
模板:
@PostMapping("$URL$")
public Result<$RETURN_TYPE$> $METHOD_NAME$(@RequestBody $REQUEST_TYPE$ request) {
    $END$
    return Result.success();
}

调试时的Expression评估 ​

调试暂停时，Alt + F8（Evaluate Expression）可以执行任意Java表达式：

java
// 查看List的内容
orders.stream().map(Order::getId).collect(Collectors.toList())

// 测试某个方法的返回值
userMapper.selectById("12345")

// 格式化输出JSON
new com.fasterxml.jackson.databind.ObjectMapper().writerWithDefaultPrettyPrinter().writeValueAsString(order)

// 检查字符串
email.contains("@") && email.endsWith(".com")

这个功能非常强大，相当于在暂停点有一个交互式Java控制台。

踩坑总结 ​

  1. IDEA内存设置。项目大了IDEA会卡，修改 Help -> Change Memory Settings，建议设到4096MB
  2. 不要过度依赖Debug。简单的逻辑用日志排查比Debug更快，Debug适合排查复杂的状态流转问题
  3. 远程调试记得用完关掉。不关的话端口暴露在外面有安全风险，而且会轻微影响性能
  4. 索引重建。IDEA偶尔会出现索引损坏导致各种奇怪问题（跳转失败、补全异常），File -> Invalidate Caches -> Invalidate and Restart 可以解决
  5. Git集成不如命令行灵活。IDEA的Git图形界面适合查看diff和做简单commit，复杂的rebase和cherry-pick建议用命令行

IDEA是一个生产力工具，熟练掌握它的调试和效率功能，开发速度能提升30%以上。建议每周学一个新快捷键，一个月后就会发现手指已经记住了。
