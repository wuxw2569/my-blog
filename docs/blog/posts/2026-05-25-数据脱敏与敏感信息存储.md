## 问题场景

安全审计来了，审计师拉了我们数据库的查询日志，发现：

1. 客服人员直接查数据库能看到用户完整手机号和身份证号
2. 日志文件里打印了用户的银行卡号
3. 前端API返回了完整的手机号（开发偷懒没做脱敏）

审计报告直接写了"存在严重的数据泄露风险"，要求两周内整改。

## 分析思路

### 1. 先定义哪些字段需要脱敏

我们梳理了所有表，标记了需要脱敏的字段：

| 数据类型 | 字段示例 | 脱敏规则 | 示例 |
|----------|----------|----------|------|
| 手机号 | user.phone | 前3后4中间* | 138****1234 |
| 身份证 | user.id_card | 前3后4中间* | 110***********1234 |
| 银行卡 | account.bank_card | 前4后4中间* | 6222 **** **** 5678 |
| 邮箱 | user.email | @前只显示首字符 | j***@example.com |
| 姓名 | user.real_name | 保留姓，名用* | 张** |
| 地址 | user.address | 保留省市，详细地址用* | 北京市朝阳区**** |

### 2. 脱敏的三层防线

**第一层：存储层（加密存储）** —— 数据库中存的是密文
**第二层：传输层（接口脱敏）** —— API返回的是脱敏后的数据
**第三层：展示层（日志脱敏）** —— 日志和控制台输出自动脱敏

### 3. 加密存储方案选择

| 方案 | 优点 | 缺点 |
|------|------|------|
| AES对称加密 | 速度快，可逆 | 需要管理密钥 |
| RSA非对称加密 | 安全性高 | 速度慢，不适合大量数据 |
| SM4国密算法 | 国内合规 | 性能略低于AES |
| 数据库自带加密 | 无侵入 | MySQL TDE只有企业版才有 |
| 单向Hash | 不可逆 | 无法还原（不适合需要展示的场景） |

**我们的选择：AES-256 + SM4 混合方案**

- 手机号、身份证号、银行卡号：AES-256 加密存储（需要展示原文时可解密）
- 登录密码：BCrypt 单向Hash（永远不需要解密）
- 关键业务数据：SM4 国密（满足国内合规要求）

## 具体实现

### 1. AES 加密工具类

```java
@Component
public class AESUtils {

    // 生产环境密钥从配置中心（如Nacos/Vault）获取，绝对不要硬编码
    @Value("${security.aes.secret-key}")
    private String secretKey;

    private static final String ALGORITHM = "AES/GCM/NoPadding";
    private static final int GCM_TAG_LENGTH = 128;
    private static final int GCM_IV_LENGTH = 12;

    public String encrypt(String plaintext) {
        if (StringUtils.isBlank(plaintext)) return plaintext;
        try {
            byte[] iv = new byte[GCM_IV_LENGTH];
            SecureRandom.getInstanceStrong().nextBytes(iv);

            SecretKeySpec keySpec = new SecretKeySpec(secretKey.getBytes(StandardCharsets.UTF_8), "AES");
            GCMParameterSpec gcmSpec = new GCMParameterSpec(GCM_TAG_LENGTH, iv);

            Cipher cipher = Cipher.getInstance(ALGORITHM);
            cipher.init(Cipher.ENCRYPT_MODE, keySpec, gcmSpec);

            byte[] encrypted = cipher.doFinal(plaintext.getBytes(StandardCharsets.UTF_8));

            // IV + 密文拼接存储
            byte[] result = new byte[iv.length + encrypted.length];
            System.arraycopy(iv, 0, result, 0, iv.length);
            System.arraycopy(encrypted, 0, result, iv.length, encrypted.length);

            return Base64.getEncoder().encodeToString(result);
        } catch (Exception e) {
            throw new RuntimeException("加密失败", e);
        }
    }

    public String decrypt(String ciphertext) {
        if (StringUtils.isBlank(ciphertext)) return ciphertext;
        try {
            byte[] decoded = Base64.getDecoder().decode(ciphertext);

            byte[] iv = Arrays.copyOfRange(decoded, 0, GCM_IV_LENGTH);
            byte[] encrypted = Arrays.copyOfRange(decoded, GCM_IV_LENGTH, decoded.length);

            SecretKeySpec keySpec = new SecretKeySpec(secretKey.getBytes(StandardCharsets.UTF_8), "AES");
            GCMParameterSpec gcmSpec = new GCMParameterSpec(GCM_TAG_LENGTH, iv);

            Cipher cipher = Cipher.getInstance(ALGORITHM);
            cipher.init(Cipher.DECRYPT_MODE, keySpec, gcmSpec);

            return new String(cipher.doFinal(encrypted), StandardCharsets.UTF_8);
        } catch (Exception e) {
            throw new RuntimeException("解密失败", e);
        }
    }
}
```

### 2. MyBatis 拦截器实现自动加解密

这是最优雅的方案：在 MyBatis 层拦截，**查询结果自动解密，写入时自动加密**。业务代码完全不用感知加解密逻辑。

**自定义注解标记需要加密的字段**：

```java
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface EncryptField {
    // 后续可扩展加密类型
}
```

**实体类标注**：

```java
@Data
public class User {
    private Long id;
    private String username;

    @EncryptField
    private String phone;

    @EncryptField
    private String idCard;

    @EncryptField
    private String bankCard;

    private String address;
}
```

**MyBatis 查询结果拦截器（解密）**：

```java
@Intercepts({
    @Signature(type = ResultSetHandler.class, method = "handleResultSets", args = {Statement.class})
})
@Component
public class DecryptInterceptor implements Interceptor {

    @Autowired
    private AESUtils aesUtils;

    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        Object result = invocation.proceed();
        if (result == null) return null;

        // 处理列表结果
        if (result instanceof List) {
            for (Object item : (List<?>) result) {
                decryptFields(item);
            }
        } else {
            decryptFields(result);
        }
        return result;
    }

    private void decryptFields(Object obj) throws IllegalAccessException {
        Class<?> clazz = obj.getClass();
        for (Field field : clazz.getDeclaredFields()) {
            if (field.isAnnotationPresent(EncryptField.class) && field.getType() == String.class) {
                field.setAccessible(true);
                String value = (String) field.get(obj);
                if (StringUtils.isNotBlank(value)) {
                    field.set(obj, aesUtils.decrypt(value));
                }
            }
        }
    }
}
```

**MyBatis 写入拦截器（加密）**：

```java
@Intercepts({
    @Signature(type = Executor.class, method = "update", args = {MappedStatement.class, Object.class})
})
@Component
public class EncryptInterceptor implements Interceptor {

    @Autowired
    private AESUtils aesUtils;

    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        Object[] args = invocation.getArgs();
        Object parameter = args[1];

        if (parameter != null) {
            encryptFields(parameter);
        }
        return invocation.proceed();
    }

    private void encryptFields(Object obj) throws IllegalAccessException {
        Class<?> clazz = obj.getClass();

        // 处理Map参数（MyBatis的@Param注解会包装成Map）
        if (obj instanceof Map) {
            Map<?, ?> map = (Map<?, ?>) obj;
            for (Object value : map.values()) {
                if (value != null && isEntityWithEncryptFields(value.getClass())) {
                    encryptFields(value);
                }
            }
            return;
        }

        if (!isEntityWithEncryptFields(clazz)) return;

        for (Field field : clazz.getDeclaredFields()) {
            if (field.isAnnotationPresent(EncryptField.class) && field.getType() == String.class) {
                field.setAccessible(true);
                String value = (String) field.get(obj);
                if (StringUtils.isNotBlank(value)) {
                    field.set(obj, aesUtils.encrypt(value));
                }
            }
        }
    }

    private boolean isEntityWithEncryptFields(Class<?> clazz) {
        for (Field field : clazz.getDeclaredFields()) {
            if (field.isAnnotationPresent(EncryptField.class)) {
                return true;
            }
        }
        return false;
    }
}
```

### 3. 接口层脱敏注解

数据库存的是密文，查询时自动解密。但API返回给前端时，不能返回完整数据，需要再次脱敏：

```java
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@JacksonAnnotationsInside
@JsonSerialize(using = SensitiveSerializer.class)
public @interface Sensitive {
    SensitiveType value();
}

public enum SensitiveType {
    PHONE, ID_CARD, BANK_CARD, EMAIL, NAME, ADDRESS
}
```

```java
public class SensitiveSerializer extends JsonSerializer<String>
    implements ContextualSerializer {

    private SensitiveType type;

    @Override
    public void serialize(String value, JsonGenerator gen, SerializerProvider provider)
            throws IOException {
        if (value == null) {
            gen.writeNull();
            return;
        }
        gen.writeString(desensitize(value, type));
    }

    private String desensitize(String value, SensitiveType type) {
        if (StringUtils.isBlank(value)) return value;
        switch (type) {
            case PHONE:
                return value.replaceAll("(\\d{3})\\d{4}(\\d{4})", "$1****$2");
            case ID_CARD:
                return value.replaceAll("(\\d{3})\\d{11}(\\w{4})", "$1***********$2");
            case BANK_CARD:
                return value.replaceAll("(\\d{4})\\d+(\\d{4})", "$1 **** **** $2");
            case EMAIL:
                return value.replaceAll("(.).+(@.+)", "$1***$2");
            case NAME:
                return value.length() <= 2
                    ? value.charAt(0) + "*"
                    : value.charAt(0) + "*".repeat(value.length() - 1);
            case ADDRESS:
                return value.length() > 6
                    ? value.substring(0, 6) + "****"
                    : "****";
            default:
                return value;
        }
    }

    @Override
    public JsonSerializer<?> createContextual(SerializerProvider prov, BeanProperty property) {
        Sensitive annotation = property.getAnnotation(Sensitive.class);
        if (annotation != null) {
            SensitiveSerializer serializer = new SensitiveSerializer();
            serializer.type = annotation.value();
            return serializer;
        }
        return this;
    }
}
```

**DTO 使用**：

```java
@Data
public class UserVO {
    private Long id;
    private String username;

    @Sensitive(SensitiveType.PHONE)
    private String phone;

    @Sensitive(SensitiveType.NAME)
    private String realName;
}
```

接口返回时自动脱敏，不需要手动调用任何方法。

### 4. 日志脱敏

用 Logback 的自定义 PatternLayout 实现日志自动脱敏：

```java
public class SensitivePatternLayout extends PatternLayout {

    private static final Map<Pattern, String> REGEX_MAP = new LinkedHashMap<>();

    static {
        // 手机号
        REGEX_MAP.put(Pattern.compile("(1[3-9]\\d)\\d{4}(\\d{4})"), "$1****$2");
        // 身份证号
        REGEX_MAP.put(Pattern.compile("(\\d{3})\\d{11}(\\w{4})"), "$1***********$2");
        // 银行卡号
        REGEX_MAP.put(Pattern.compile("(\\d{4})\\d{8,12}(\\d{4})"), "$1****$2");
    }

    @Override
    public String doLayout(ILoggingEvent event) {
        String message = super.doLayout(event);
        for (Map.Entry<Pattern, String> entry : REGEX_MAP.entrySet()) {
            message = entry.getKey().matcher(message).replaceAll(entry.getValue());
        }
        return message;
    }
}
```

```xml
<!-- logback-spring.xml -->
<appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
    <encoder class="ch.qos.logback.core.encoder.LayoutWrappingEncoder">
        <layout class="com.example.log.SensitivePatternLayout">
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </layout>
    </encoder>
</appender>
```

## 踩坑总结

1. **加密密钥管理是核心**。密钥不能硬编码在代码里，不能提交到Git。我们用 Nacos 配置中心 + 环境变量，生产环境用 KMS（密钥管理服务）。
2. **加密后的字段无法建索引**。手机号加了AES加密后，原来的 `WHERE phone = ?` 查询就走不了索引了。我们的方案是：加密字段同时存一份 Hash 值用于查询匹配。
3. **存量数据需要迁移**。上线加密方案后，历史数据还是明文。我们写了个迁移脚本，分批把历史数据加密，迁移期间接口要兼容新旧两种格式。
4. **脱敏规则要统一**。开发A用正则脱敏，开发B用字符串截取，规则不一致。一定要在工具类中统一封装，不要让每个开发者自己写脱敏逻辑。
5. **日志脱敏有性能开销**。正则匹配每条日志消息，日志量大时有可感知的CPU消耗。如果性能敏感，可以只对特定包的日志启用脱敏。
6. **合规是底线**。个人隐私数据（手机号、身份证）是《个人信息保护法》明确保护的，泄露可能面临巨额罚款。这不是技术优化，是法律要求。
