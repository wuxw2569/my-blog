### 问题场景

接口文档是前后端协作的重灾区。我们团队经历了三个阶段：Word文档（手动维护，永远过期） -> Swagger2（功能够了但UI丑、配置复杂） -> SpringDoc + Knife4j（最终方案）。这篇文章记录了我们对Swagger工具链的选型和落地过程。

### SpringDoc OpenAPI 3 配置

Swagger2已经停止维护，Springfox也不再更新。当前推荐使用SpringDoc（支持OpenAPI 3.0）。

**引入依赖**：

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.3.0</version>
</dependency>
```

**基础配置类**：

```java
@Configuration
public class OpenApiConfig {

    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
                .info(new Info()
                        .title("订单服务 API")
                        .version("1.0.0")
                        .description("订单服务对外接口文档")
                        .contact(new Contact()
                                .name("后端团队")
                                .email("backend@ourcompany.com")))
                .addSecurityItem(new SecurityRequirement().addList("Bearer"))
                .components(new Components()
                        .addSecuritySchemes("Bearer",
                                new SecurityScheme()
                                        .type(SecurityScheme.Type.HTTP)
                                        .scheme("bearer")
                                        .bearerFormat("JWT")));
    }
}
```

**application.yml 配置**：

```yaml
springdoc:
  api-docs:
    path: /v3/api-docs
  swagger-ui:
    path: /swagger-ui.html
    # 按tags分组排序
    tags-sorter: alpha
    operations-sorter: method
  # 扫描指定包（不要扫描所有Controller，会很慢）
  packages-to-scan: com.ourcompany.order.controller
  # 排除error等系统接口
  paths-to-exclude: /error,/actuator/**
```

启动后访问 `http://localhost:8080/swagger-ui.html` 就能看到接口文档。

```java

public class CreateOrderRequest {

    @Schema(description = "购买数量", minimum = "1", maximum = "99", example = "2")
```

### Knife4j 增强

Knife4j是国人开发的Swagger增强工具，UI更好看，功能更强。

**引入依赖**（替换掉原生的springdoc-ui）：

```xml
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    <version>4.4.0</version>
</dependency>
```

**增强配置**：

```java
@Configuration
public class Knife4jConfig {

    @Bean
    public OpenApiCustomiser openApiCustomiser() {
        return openApi -> {
            // 全局响应示例
            openApi.getPaths().forEach((path, pathItem) -> {
                pathItem.readOperations().forEach(operation -> {
                    // 给所有接口加上统一的错误响应
                    operation.getResponses().addApiResponse("500",
                            new ApiResponse().description("服务器内部错误")
                                    .content(new Content().addMediaType("application/json",
                                            new MediaType().schema(new Schema<>().$ref("#/components/schemas/Result")))));
                });
            });
        };
    }
}
```

**Knife4j相比原生Swagger的优势**：
- 离线导出文档（Markdown/Word/PDF）
- 接口搜索功能
- 全局参数管理（Token等）
- 更好的分组管理
- Mock数据支持

访问地址：`http://localhost:8080/doc.html`

### 接口文档与代码同步

文档最大的问题是"代码改了文档没更新"。我们的做法：

**1. DTO和VO就是文档源**

不要用Map或者裸Object返回，定义明确的DTO和VO，用注解标注字段含义。代码就是文档，改代码就改了文档。

**2. CI中检查接口文档是否可访问**

```bash
# 在CI流水线中验证接口文档是否正常生成
curl -f http://localhost:8080/v3/api-docs | python -m json.tool > /dev/null
if [ $? -ne 0 ]; then
    echo "接口文档生成失败！"
    exit 1
fi
```

**3. 导出接口文档到团队wiki**

```bash
# 用knife4j的增强文档接口导出Markdown
curl http://localhost:8080/v3/api-docs > api-docs.json

# 使用swagger-codegen生成Markdown文档
java -jar swagger-codegen-cli.jar generate \
    -i api-docs.json \
    -l markdown \
    -o ./docs/api/
```

### Mock数据

前端联调时经常等后端接口。用Knife4j的Mock功能可以在后端接口没写完时就提供Mock数据。

**方式1：注解Mock**

```java
@Schema(description = "订单详情")
public class OrderVO {

    @Schema(description = "订单ID", example = "ORD20260525001")
    private String orderId;

    @Schema(description = "商品名称", example = "测试商品A")
    private String skuName;

    @Schema(description = "订单金额", example = "299.00")
    private BigDecimal amount;

    @Schema(description = "订单状态", example = "PAID",
            allowableValues = {"PENDING", "PAID", "SHIPPED", "COMPLETED", "CANCELLED"})
    private String status;
}
```

`example` 字段的值会作为Mock数据返回。

**方式2：全局Mock**

在Knife4j的文档页面上，可以对接口开启Mock，选择"自定义Mock"或"系统Mock"。

**方式3：本地Mock Server**

```yaml
# application-dev.yml
springdoc:
  # 开发环境启用Mock
  swagger-ui:
    # 前端可以通过这个地址获取Mock数据
    url: /v3/api-docs
```

### 生产环境关闭Swagger

生产环境不要暴露接口文档：

```yaml
# application-prod.yml
springdoc:
  api-docs:
    enabled: false
  swagger-ui:
    enabled: false
```

或者用Profile条件控制Bean注册：

```java
@Configuration
@Profile({"dev", "test"})  // 只在开发和测试环境启用
public class OpenApiConfig {
    // ...
}
```

### 踩坑总结

1. **从Swagger2迁移到SpringDoc**：注解不兼容，`@Api` 变 `@Tag`，`@ApiOperation` 变 `@Operation`，需要全局替换。可以用正则批量处理
2. **接口文档生成慢**：扫描包范围太大。`packages-to-scan` 要精确到controller包，不要扫整个项目
3. **枚举类型的展示**：用 `allowableValues` 标注枚举的可选值，前端不用翻代码就知道传什么
4. **分页接口的参数**：`@ParameterObject` 注解可以把Pageable等对象的字段展开显示，比传一个JSON对象更直观
5. **返回值不要用泛型擦除**：`Result<T>` 在文档中可能显示为 `Result<Object>`。用 `@Schema(implementation = XxxVO.class)` 明确指定具体类型

接口文档自动化的核心原则：文档即代码，改代码就是改文档，消除手动维护的成本。
