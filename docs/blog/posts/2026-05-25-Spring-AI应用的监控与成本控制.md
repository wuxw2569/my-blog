## 问题场景

客服 AI 系统上线两个月后，财务部门找过来了："这个月 AI 调用费用比预算高了 3 倍，你们能不能说清楚钱花在哪了？"

我们翻了一圈日志，发现根本说不清楚。因为之前只记录了"调用成功/失败"，没有记录每次调用消耗了多少 Token、用的什么模型、对应哪个业务场景。更尴尬的是，有几次 DashScope API 响应特别慢，用户以为系统挂了疯狂重试，Token 白白烧了不少。

这件事之后，我们痛定思痛，花了两周搭了一套完整的 AI 监控体系。

## 分析思路

AI 应用的监控跟传统应用不一样，除了常规的 QPS、延迟、错误率，还需要关注：

1. **Token 消耗**：每个接口、每个用户、每个模型分别消耗了多少
2. **调用链路**：一次 RAG 请求经过了哪些步骤（Embedding 检索 + 模型生成）
3. **成本预警**：日/周/月消耗达到阈值时自动告警
4. **质量指标**：用户满意度、对话轮次、Function Calling 成功率

## 具体实现

### 第一步：Token 统计与埋点

```java
/**
 * AI 调用指标收集器
 * 作为 Advisor 注入到 ChatClient 调用链路中
 */
@Component
@Slf4j
public class AiMetricsAdvisor implements CallAroundAdvisor {

    private final MeterRegistry meterRegistry;
    private final TokenUsageRepository tokenUsageRepository;

    public AiMetricsAdvisor(MeterRegistry meterRegistry, TokenUsageRepository tokenUsageRepository) {
        this.meterRegistry = meterRegistry;
        this.tokenUsageRepository = tokenUsageRepository;
    }

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
        long startTime = System.currentTimeMillis();

        // 从上下文获取业务标识
        String bizType = request.adviseContext().getOrDefault("bizType", "unknown");
        String userId = request.adviseContext().getOrDefault("userId", "anonymous");

        try {
            AdvisedResponse response = chain.nextAroundCall(request);

            long duration = System.currentTimeMillis() - startTime;

            // 记录 Token 使用量
            ChatResponse chatResponse = response.response();
            if (chatResponse.getMetadata() != null && chatResponse.getMetadata().getUsage() != null) {
                Usage usage = chatResponse.getMetadata().getUsage();
                recordMetrics(bizType, userId, usage, duration, true);
            }

            return response;
        } catch (Exception e) {
            long duration = System.currentTimeMillis() - startTime;
            recordFailureMetrics(bizType, userId, duration, e);
            throw e;
        }
    }

    private void recordMetrics(String bizType, String userId, Usage usage,
                                long duration, boolean success) {
        int promptTokens = (int) usage.getPromptTokens();
        int completionTokens = (int) usage.getCompletionTokens();
        int totalTokens = promptTokens + completionTokens;

        // Prometheus 指标
        meterRegistry.counter("ai.requests.total",
                "bizType", bizType, "success", String.valueOf(success)).increment();
        meterRegistry.counter("ai.tokens.total",
                "bizType", bizType, "type", "prompt").increment(promptTokens);
        meterRegistry.counter("ai.tokens.total",
                "bizType", bizType, "type", "completion").increment(completionTokens);
        meterRegistry.timer("ai.requests.duration", "bizType", bizType)
                .record(duration, TimeUnit.MILLISECONDS);

        // 持久化到数据库（异步，不影响主流程）
        TokenUsageRecord record = new TokenUsageRecord();
        record.setBizType(bizType);
        record.setUserId(userId);
        record.setPromptTokens(promptTokens);
        record.setCompletionTokens(completionTokens);
        record.setTotalTokens(totalTokens);
        record.setDurationMs(duration);
        record.setCreatedAt(Instant.now());
        tokenUsageRepository.saveAsync(record);

        log.info("【AI指标】bizType={}, user={}, prompt={}, completion={}, total={}, duration={}ms",
                bizType, userId, promptTokens, completionTokens, totalTokens, duration);
    }

    private void recordFailureMetrics(String bizType, String userId,
                                       long duration, Exception e) {
        meterRegistry.counter("ai.requests.total",
                "bizType", bizType, "success", "false").increment();
        meterRegistry.timer("ai.requests.duration", "bizType", bizType)
                .record(duration, TimeUnit.MILLISECONDS);
        log.error("【AI调用失败】bizType={}, user={}, duration={}ms, error={}",
                bizType, userId, duration, e.getMessage());
    }

    @Override
    public String getName() {
        return "AiMetricsAdvisor";
    }

    @Override
    public int getOrder() {
        return Ordered.LOWEST_PRECEDENCE;
    }
}
```

### 第二步：在业务代码中传入上下文

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class MonitoredChatService {

    private final ChatClient chatClient;

    /**
     * 带业务标识的对话调用
     * bizType 用于区分不同业务场景，方便成本归因
     */
    public String chat(String userId, String bizType, String message) {
        return chatClient.prompt()
                .user(message)
                .advisors(a -> a
                        .param("bizType", bizType)
                        .param("userId", userId))
                .call()
                .content();
    }
}

@RestController
@RequestMapping("/api/chat")
@RequiredArgsConstructor
class ChatController {

    private final MonitoredChatService chatService;

    @PostMapping("/customer-service")
    public Map<String, Object> customerService(@RequestBody ChatRequest req,
                                                @RequestHeader("X-User-Id") String userId) {
        // bizType = customer-service 会体现在所有监控指标中
        String answer = chatService.chat(userId, "customer-service", req.message());
        return Map.of("answer", answer);
    }

    @PostMapping("/knowledge-qa")
    public Map<String, Object> knowledgeQa(@RequestBody ChatRequest req,
                                            @RequestHeader("X-User-Id") String userId) {
        String answer = chatService.chat(userId, "knowledge-qa", req.message());
        return Map.of("answer", answer);
    }

    public record ChatRequest(String message) {}
}
```

### 第三步：Token 使用量统计表

```sql
CREATE TABLE token_usage (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    biz_type VARCHAR(32) NOT NULL COMMENT '业务类型',
    user_id VARCHAR(64) NOT NULL COMMENT '用户ID',
    model VARCHAR(32) DEFAULT '' COMMENT '模型名称',
    prompt_tokens INT NOT NULL COMMENT '输入Token数',
    completion_tokens INT NOT NULL COMMENT '输出Token数',
    total_tokens INT NOT NULL COMMENT '总Token数',
    duration_ms BIGINT NOT NULL COMMENT '耗时(毫秒)',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_biz_date (biz_type, created_at),
    INDEX idx_user_date (user_id, created_at)
);
```

### 第四步：成本统计与预警

```java
@Service
@Slf4j
@RequiredArgsConstructor
public class CostMonitorService {

    private final JdbcTemplate jdbcTemplate;
    private final NotificationService notificationService;

    // DashScope 各模型的价格（元/千Token），需要定期更新
    private static final Map<String, BigDecimal> MODEL_PRICE = Map.of(
            "qwen-turbo", new BigDecimal("0.002"),
            "qwen-plus", new BigDecimal("0.004"),
            "qwen-max", new BigDecimal("0.020"),
            "text-embedding-v2", new BigDecimal("0.0007")
    );

    // 预算阈值
    private static final BigDecimal DAILY_BUDGET = new BigDecimal("500");   // 日预算 500 元
    private static final BigDecimal MONTHLY_BUDGET = new BigDecimal("10000"); // 月预算 1 万元

    /**
     * 计算今日费用
     */
    public CostSummary getTodayCost() {
        String sql = """
                SELECT biz_type,
                       SUM(prompt_tokens) as total_prompt,
                       SUM(completion_tokens) as total_completion,
                       SUM(total_tokens) as total_tokens,
                       COUNT(*) as call_count,
                       AVG(duration_ms) as avg_duration
                FROM token_usage
                WHERE DATE(created_at) = CURDATE()
                GROUP BY biz_type
                """;

        List<Map<String, Object>> rows = jdbcTemplate.queryForList(sql);
        BigDecimal totalCost = BigDecimal.ZERO;

        List<BizCost> bizCosts = new ArrayList<>();
        for (Map<String, Object> row : rows) {
            String bizType = (String) row.get("biz_type");
            int totalTokens = ((Number) row.get("total_tokens")).intValue();
            int callCount = ((Number) row.get("call_count")).intValue();
            long avgDuration = ((Number) row.get("avg_duration")).longValue();

            // 简化计算：按 qwen-plus 价格估算
            BigDecimal cost = MODEL_PRICE.getOrDefault("qwen-plus", BigDecimal.ZERO)
                    .multiply(new BigDecimal(totalTokens))
                    .divide(new BigDecimal(1000), 4, RoundingMode.HALF_UP);

            totalCost = totalCost.add(cost);
            bizCosts.add(new BizCost(bizType, callCount, totalTokens, cost, avgDuration));
        }

        return new CostSummary(LocalDate.now(), totalCost, bizCosts);
    }

    /**
     * 每小时检查预算预警
     */
    @Scheduled(cron = "0 0 * * * ?")
    public void checkBudget() {
        CostSummary todayCost = getTodayCost();

        // 日预算预警
        BigDecimal usagePercent = todayCost.totalCost()
                .divide(DAILY_BUDGET, 2, RoundingMode.HALF_UP)
                .multiply(new BigDecimal(100));

        if (usagePercent.compareTo(new BigDecimal("80")) >= 0) {
            String level = usagePercent.compareTo(new BigDecimal("100")) >= 0 ? "严重" : "警告";
            String message = String.format("【AI费用%s】今日费用 %.2f 元，已达预算的 %.0f%%",
                    level, todayCost.totalCost(), usagePercent);

            log.warn(message);
            notificationService.sendAlert(message);

            // 超出预算时自动降级到便宜模型
            if (usagePercent.compareTo(new BigDecimal("100")) >= 0) {
                log.warn("【成本控制】已超出日预算，自动切换到 qwen-turbo");
                // 通过配置中心切换模型
            }
        }

        // 月预算检查
        checkMonthlyBudget();
    }

    private void checkMonthlyBudget() {
        String sql = """
                SELECT COALESCE(SUM(total_tokens), 0) as month_tokens
                FROM token_usage
                WHERE YEAR(created_at) = YEAR(CURDATE())
                  AND MONTH(created_at) = MONTH(CURDATE())
                """;
        Long monthTokens = jdbcTemplate.queryForObject(sql, Long.class);
        if (monthTokens == null) return;

        BigDecimal monthCost = MODEL_PRICE.getOrDefault("qwen-plus", BigDecimal.ZERO)
                .multiply(new BigDecimal(monthTokens))
                .divide(new BigDecimal(1000), 4, RoundingMode.HALF_UP);

        BigDecimal usagePercent = monthCost
                .divide(MONTHLY_BUDGET, 2, RoundingMode.HALF_UP)
                .multiply(new BigDecimal(100));

        if (usagePercent.compareTo(new BigDecimal("80")) >= 0) {
            String message = String.format("【月度AI费用预警】本月费用 %.2f 元，已达预算的 %.0f%%",
                    monthCost, usagePercent);
            log.warn(message);
            notificationService.sendAlert(message);
        }
    }

    public record CostSummary(LocalDate date, BigDecimal totalCost, List<BizCost> bizCosts) {}
    public record BizCost(String bizType, int callCount, int totalTokens,
                           BigDecimal cost, long avgDurationMs) {}
}
```

### 第五步：监控大盘（Grafana 配置）

```yaml
# application-monitor.yml
management:
  endpoints:
    web:
      exposure:
        include: health, metrics, prometheus
  metrics:
    tags:
      application: ai-customer-service
    distribution:
      percentiles-histogram:
        ai.requests.duration: true
      sla:
        ai.requests.duration: 200ms, 500ms, 1s, 3s
```

Grafana 关键看板指标：
- **QPS 面板**：`rate(ai_requests_total[5m])` 按 bizType 分组
- **Token 消耗面板**：`sum(rate(ai_tokens_total[1h]))` 按 type 分组
- **延迟面板**：`histogram_quantile(0.99, ai_requests_duration_bucket)`
- **费用面板**：基于 Token 计数器计算实时费用

```java
/**
 * AI 调用日志格式规范
 * 所有 AI 相关日志统一格式，方便 ELK 检索
 */
public class AiLogFormat {

    /**
     * 请求开始日志
     */
    public static String requestStart(String bizType, String userId, String input) {
        return String.format("【AI请求开始】 bizType=%s | userId=%s | inputLen=%d | input=%s",
                bizType, userId, input.length(),
                input.substring(0, Math.min(200, input.length())));
    }

    /**
     * 请求完成日志
     */
    public static String requestEnd(String bizType, String userId,
                                     int promptTokens, int completionTokens,
                                     long durationMs, boolean success) {
        return String.format(
                "【AI请求完成】 bizType=%s | userId=%s | promptTokens=%d | completionTokens=%d | total=%d | duration=%dms | success=%s",
                bizType, userId, promptTokens, completionTokens,
                promptTokens + completionTokens, durationMs, success);
    }

    /**
     * 异常日志
     */
    public static String error(String bizType, String userId, String errorType, String message) {
        return String.format("【AI异常】 bizType=%s | userId=%s | errorType=%s | message=%s",
                bizType, userId, errorType, message);
    }
}
```

## 踩坑总结

1. **Token 统计跟 DashScope 对账有差异**：我们自己统计的 Token 数跟 DashScope 控制台显示的有 5-10% 的偏差。原因是部分统计维度（比如图片 Token）我们没算进去。最终以 DashScope 控制台数据为准，自己的统计作为实时监控用。

2. **异步写数据库别用同步方式**：一开始 Token 统计是同步写 MySQL 的，结果 AI 响应延迟增加了 20ms。改成异步写入（用 `@Async` 或消息队列）后解决了。但要注意队列不能积压太多，否则监控数据延迟。

3. **Grafana 指标的基数爆炸**：`ai.tokens.total` 这个指标按 bizType 分标签，如果 bizType 是动态的（比如用用户 ID），Prometheus 的时间序列会爆炸。bizType 只用固定的几个值（customer-service, knowledge-qa 等），用户维度的统计放到数据库里查。

4. **DashScope 的并发限制不体现在 Token 消耗里**：DashScope 除了 Token 计费，还有 QPS 限制。我们遇到过"Token 预算没超但请求被限流"的情况。需要同时监控 QPS 指标。

5. **成本预警的响应时效**：每小时检查一次预算是不够的。如果有异常调用（比如爬虫），一小时内就能烧掉大量预算。建议在 Token 统计时就做实时检查，超阈值立即告警。

6. **日志中的隐私合规**：用户的输入内容不能原样写到日志里，尤其是 ELK 这种集中式日志系统。要做脱敏处理：截断到前 100 字符、遮蔽手机号等 PII 信息。GDPR/个人信息保护法要求日志中的个人信息 72 小时内删除。
