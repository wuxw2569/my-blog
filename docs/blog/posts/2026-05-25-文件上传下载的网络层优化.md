## 前言

做过企业级系统的同学应该都有过这种经历：业务方说"我要上传一个 2GB 的 Excel 报表"，你用 `MultipartFile` 一接，结果 OOM 了。或者："我要下载一份审计日志，大概 5GB"，你用 `IOUtils.copy()` 一刷，结果下载到 80% 网络抖了一下，前功尽弃。

这两个场景分别对应了文件传输的两个核心问题：**大文件上传**和**大文件下载**。这篇文章分享我们在实际项目中总结的网络层优化方案。

## 问题场景

我们的文档管理系统需要支持：

1. 员工上传 50MB~2GB 的文件（设计图纸、视频素材等）
2. 管理员导出 100MB~5GB 的审计日志
3. 支持上传中断后断点续传
4. 下载速度要快，用 CDN 加速

初始方案的痛点：

- 直接用 `MultipartFile`，超过 200MB 就 OOM
- 单次上传，网络断了要重传
- 下载不支持断点，断了从头来
- 文件不走 CDN，全部经过应用服务器

## 具体实现

### 一、大文件分片上传

核心思路：前端把大文件切成固定大小的分片（比如 5MB 一片），逐片上传到后端，全部上传完成后后端合并。

**后端接口设计**：

```java
@RestController
@RequestMapping("/api/file")
@Slf4j
public class FileUploadController {

    @Autowired
    private FileService fileService;

    /**
     * 1. 初始化分片上传
     */
    @PostMapping("/upload/init")
    public Result<UploadInitVO> initUpload(@RequestBody UploadInitDTO dto) {
        // dto 包含：fileName, fileSize, chunkSize, md5(整个文件的MD5)
        UploadInitVO vo = fileService.initUpload(dto);
        return Result.success(vo);
        // 返回：uploadId, 已上传的分片列表（用于断点续传）
    }

    /**
     * 2. 上传单个分片
     */
    @PostMapping("/upload/chunk")
    public Result<Void> uploadChunk(
            @RequestParam("uploadId") String uploadId,
            @RequestParam("chunkIndex") int chunkIndex,
            @RequestParam("chunk") MultipartFile chunk) {
        fileService.uploadChunk(uploadId, chunkIndex, chunk);
        return Result.success();
    }

    /**
     * 3. 合并分片
     */
    @PostMapping("/upload/merge")
    public Result<String> mergeChunks(@RequestBody MergeChunksDTO dto) {
        String fileUrl = fileService.mergeChunks(dto.getUploadId());
        return Result.success(fileUrl);
    }
}
```

**分片上传核心实现**：

```java
@Service
@Slf4j
public class FileService {

    // 分片临时目录
    private static final String CHUNK_TEMP_DIR = "/data/upload/chunks/";

    /**
     * 初始化上传：生成 uploadId，创建分片目录
     */
    public UploadInitVO initUpload(UploadInitDTO dto) {
        // 用文件 MD5 作为幂等键，相同文件秒传
        String existingFile = checkFileExists(dto.getMd5());
        if (existingFile != null) {
            log.info("【秒传】文件已存在, md5={}", dto.getMd5());
            UploadInitVO vo = new UploadInitVO();
            vo.setUploadId(null);  // null 表示秒传
            vo.setFileUrl(existingFile);
            vo.setCompleted(true);
            return vo;
        }

        // 生成 uploadId（UUID 或雪花 ID）
        String uploadId = UUID.randomUUID().toString().replace("-", "");
        String chunkDir = CHUNK_TEMP_DIR + uploadId;
        new File(chunkDir).mkdirs();

        // 查询已上传的分片（断点续传）
        List<Integer> uploadedChunks = getUploadedChunks(chunkDir);

        UploadInitVO vo = new UploadInitVO();
        vo.setUploadId(uploadId);
        vo.setUploadedChunks(uploadedChunks);
        vo.setCompleted(false);
        return vo;
    }

    /**
     * 上传分片：直接写磁盘，不进 JVM 堆
     */
    public void uploadChunk(String uploadId, int chunkIndex, MultipartFile chunk) {
        String chunkPath = CHUNK_TEMP_DIR + uploadId + "/" + chunkIndex;
        try {
            // 用 transferTo 直接写磁盘，避免占用堆内存
            chunk.transferTo(new File(chunkPath));
            log.info("【分片上传】uploadId={}, chunkIndex={}, size={}",
                    uploadId, chunkIndex, chunk.getSize());
        } catch (IOException e) {
            log.error("【分片上传失败】uploadId={}, chunkIndex={}", uploadId, chunkIndex, e);
            throw new BizException("分片上传失败");
        }
    }

    /**
     * 合并分片：顺序读取分片文件，流式写入目标文件
     */
    public String mergeChunks(String uploadId) {
        String chunkDir = CHUNK_TEMP_DIR + uploadId;
        File dir = new File(chunkDir);
        File[] chunks = dir.listFiles();
        if (chunks == null || chunks.length == 0) {
            throw new BizException("没有找到分片文件");
        }

        // 按分片序号排序
        Arrays.sort(chunks, Comparator.comparingInt(
                f -> Integer.parseInt(f.getName())));

        // 合并后的文件路径
        String targetPath = "/data/upload/files/" + uploadId + getFileExtension(chunks[0]);

        try (FileOutputStream fos = new FileOutputStream(targetPath)) {
            byte[] buffer = new byte[8192];
            for (File chunk : chunks) {
                try (FileInputStream fis = new FileInputStream(chunk)) {
                    int len;
                    while ((len = fis.read(buffer)) != -1) {
                        fos.write(buffer, 0, len);
                    }
                }
            }
        } catch (IOException e) {
            log.error("【分片合并失败】uploadId={}", uploadId, e);
            throw new BizException("分片合并失败");
        }

        // 清理分片临时文件
        for (File chunk : chunks) {
            chunk.delete();
        }
        dir.delete();

        return targetPath;
    }
}
```

**Spring Boot 文件大小配置**：

```yaml
# application.yml
spring:
  servlet:
    multipart:
      # 单个分片最大 10MB
      max-file-size: 10MB
      # 单次请求最大 10MB（因为是分片上传，单次请求就是一片）
      max-request-size: 10MB
      # 写入临时文件的阈值：超过 1MB 就写磁盘，不占堆
      file-size-threshold: 1MB
```

### 二、流式下载 + 断点续传

大文件下载的核心是**流式传输**：不要把整个文件读进内存，而是一边读一边写。

同时支持 HTTP Range 请求，实现断点续传。

```java
@GetMapping("/download/{fileId}")
public void download(@PathVariable String fileId,
                     HttpServletRequest request,
                     HttpServletResponse response) throws IOException {
    File file = getRealFile(fileId);
    long fileLength = file.length();

    // 解析 Range 请求头
    String range = request.getHeader("Range");
    long start = 0;
    long end = fileLength - 1;

    if (range != null) {
        // Range: bytes=1024-2048 或 Range: bytes=1024-
        range = range.replace("bytes=", "");
        String[] parts = range.split("-");
        start = Long.parseLong(parts[0]);
        if (parts.length > 1 && !parts[1].isEmpty()) {
            end = Long.parseLong(parts[1]);
        }

        // 设置 206 Partial Content
        response.setStatus(HttpServletResponse.SC_PARTIAL_CONTENT);
    }

    long contentLength = end - start + 1;

    // 设置响应头
    response.setContentType("application/octet-stream");
    response.setHeader("Content-Disposition",
            "attachment; filename=\"" + URLEncoder.encode(file.getName(), "UTF-8") + "\"");
    response.setHeader("Accept-Ranges", "bytes");
    response.setHeader("Content-Length", String.valueOf(contentLength));

    if (range != null) {
        response.setHeader("Content-Range",
                "bytes " + start + "-" + end + "/" + fileLength);
    }

    // 流式写入
    try (RandomAccessFile raf = new RandomAccessFile(file, "r");
         OutputStream out = response.getOutputStream()) {
        raf.seek(start);
        byte[] buffer = new byte[8192];
        long remaining = contentLength;
        int len;

        while (remaining > 0) {
            len = (int) Math.min(buffer.length, remaining);
            int read = raf.read(buffer, 0, len);
            if (read == -1) break;
            out.write(buffer, 0, read);
            remaining -= read;
        }
        out.flush();
    }
}
```

前端配合断点续传，可以用 `Blob.slice()` 结合 `fetch` 实现。

### 三、CDN 加速

文件上传到应用服务器后，同步到对象存储（如阿里云 OSS），然后通过 CDN 分发。

```java
@Service
@Slf4j
public class CdnSyncService {

    @Autowired
    private OSSClient ossClient;

    private static final String BUCKET = "your-bucket";
    private static final String CDN_DOMAIN = "https://cdn.example.com";

    /**
     * 上传完成后异步同步到 OSS
     */
    @Async
    public void syncToOss(String localPath, String objectKey) {
        try {
            File file = new File(localPath);
            ossClient.putObject(BUCKET, objectKey, file);
            log.info("【CDN同步完成】key={}, size={}", objectKey, file.length());

            // 同步完成后可以删除本地文件，节省磁盘
            // file.delete();
        } catch (Exception e) {
            log.error("【CDN同步失败】key={}", objectKey, e);
            // 重试逻辑...
        }
    }

    /**
     * 获取 CDN 下载链接
     */
    public String getCdnUrl(String objectKey) {
        return CDN_DOMAIN + "/" + objectKey;
    }
}
```

### 四、上传进度回调

如果需要实时展示上传进度，可以用 SSE（Server-Sent Events）：

```java
@GetMapping("/upload/progress/{uploadId}")
public SseEmitter getProgress(@PathVariable String uploadId) {
    SseEmitter emitter = new SseEmitter(300_000L); // 5 分钟超时

    // 注册进度监听
    UploadProgressListener listener = progress -> {
        try {
            emitter.send(SseEmitter.event()
                    .name("progress")
                    .data(progress));
        } catch (IOException e) {
            emitter.completeWithError(e);
        }
    };
    progressRegistry.register(uploadId, listener);

    emitter.onCompletion(() -> progressRegistry.unregister(uploadId));
    emitter.onTimeout(() -> progressRegistry.unregister(uploadId));

    return emitter;
}
```

## 踩坑总结

| 坑 | 表现 | 解决方案 |
|----|------|----------|
| MultipartFile 直接读大文件 | OOM | 分片上传 + `transferTo` 写磁盘 |
| 分片上传网络中断 | 要从头开始 | init 接口返回已上传分片列表 |
| 文件名乱码 | 下载的文件名是乱码 | `URLEncoder.encode` + 系统编码 |
| Nginx 限制请求体大小 | 413 Request Entity Too Large | `client_max_body_size 50m` |
| 合并分片时磁盘满 | 合并失败，分片也丢了 | 先检查磁盘空间再合并 |
| CDN 缓存不刷新 | 文件更新后用户下载旧版本 | OSS 回调 + CDN 刷新 |
| RandomAccessFile 不关闭 | 文件句柄泄漏 | try-with-resources |

**Nginx 上传相关配置**：

```nginx
client_max_body_size 50m;          # 分片大小，不是总文件大小
client_body_buffer_size 128k;      # 请求体缓冲区
proxy_read_timeout 300s;           # 大分片上传可能慢
proxy_send_timeout 300s;
proxy_connect_timeout 10s;
```

整个方案上线后，2GB 文件的上传成功率从之前的约 60% 提升到了 99%+，下载也因为 CDN 加速快了很多。最大的教训是：**大文件传输一定不能经过应用服务器的堆内存**，分片 + 流式是基本原则。
