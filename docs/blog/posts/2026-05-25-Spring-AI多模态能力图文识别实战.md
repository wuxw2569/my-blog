## 问题场景

电商客服系统有了文字对话能力后，业务方又提了新需求：用户上传商品图片，系统自动识别是什么商品、读取图片上的文字（比如快递单号、商品标签）。

以前这类需求靠 OCR 库（Tesseract、百度 OCR API）来做，但 OCR 只能识别文字，理解不了图片内容。现在多模态大模型（比如通义千问 VL）既能看图又能理解，一个 API 搞定图片理解和 OCR。

但接入后发现，从"能用"到"好用"中间差了不少东西：图片格式处理、Prompt 设计、超大图片处理、与业务流程集成。

## 分析思路

Spring AI 的多模态能力通过 `Media` 对象实现，可以将图片作为消息的一部分发给模型。通义千问 VL 支持：
- 图片内容理解（描述图片里有什么）
- OCR（识别图片上的文字）
- 图片问答（针对图片内容提问）

我们的落地方案分三步：
1. 基础图片理解能力验证
2. 业务场景优化（Prompt 工程）
3. 与现有系统集成

## 具体实现

### 第一步：基础图片理解

```java
@Service
@Slf4j
@RequiredArgsConstructor
public class ImageUnderstandingService {

    private final ChatModel chatModel;

    /**
     * 识别图片内容
     * @param imageUrl 图片的 URL 地址
     * @param question 对图片的提问
     */
    public String understandImage(String imageUrl, String question) {
        log.info("【图片理解】URL: {}, 问题: {}", imageUrl, question);

        // 构建包含图片的消息
        UserMessage message = UserMessage.builder()
                .text(question)
                .media(new Media(MimeTypeUtils.IMAGE_PNG, URI.create(imageUrl)))
                .build();

        Prompt prompt = new Prompt(List.of(
                new SystemMessage("你是一个图片分析助手。请仔细观察图片并准确回答问题。回答要简洁明确。"),
                message
        ));

        ChatResponse response = chatModel.call(prompt);
        String answer = response.getResult().getOutput().getContent();

        log.info("【图片理解完成】Token使用: {}",
                response.getMetadata().getUsage().getTotalTokens());
        return answer;
    }

    /**
     * 支持 Base64 图片输入
     * 适用于用户直接上传图片的场景
     */
    public String understandBase64Image(String base64Data, String mediaType, String question) {
        byte[] imageBytes = Base64.getDecoder().decode(base64Data);
        ByteArrayResource resource = new ByteArrayResource(imageBytes);

        UserMessage message = UserMessage.builder()
                .text(question)
                .media(new Media(MimeTypeUtils.parseMimeType(mediaType), resource))
                .build();

        Prompt prompt = new Prompt(message);
        return chatModel.call(prompt).getResult().getOutput().getContent();
    }
}
```

### 第二步：商品识别业务场景

```java
@Service
@Slf4j
@RequiredArgsConstructor
public class ProductRecognitionService {

    private final ChatModel chatModel;
    private final ProductService productService;

    /**
     * 商品图片识别
     * 识别商品类型、品牌、规格，然后匹配商品库
     */
    public ProductRecognizeResult recognizeProduct(MultipartFile imageFile) {
        log.info("【商品识别】文件: {}, 大小: {}KB",
                imageFile.getOriginalFilename(), imageFile.getSize() / 1024);

        // 图片预处理：压缩过大的图片
        byte[] imageBytes = preprocessImage(imageFile);

        UserMessage message = UserMessage.builder()
                .text("""
                        请识别这张商品图片，输出以下信息（JSON格式）：
                        {
                            "category": "商品大类（如：服装、食品、电子产品）",
                            "subcategory": "商品子类（如：T恤、饼干、耳机）",
                            "brand": "品牌名（如果能识别到）",
                            "color": "主色调",
                            "description": "简短描述",
                            "keywords": ["搜索关键词1", "关键词2"]
                        }
                        如果无法识别，对应字段填 null。
                        """)
                .media(new Media(MimeTypeUtils.IMAGE_JPEG, new ByteArrayResource(imageBytes)))
                .build();

        String result = chatModel.call(new Prompt(message))
                .getResult().getOutput().getContent();

        // 解析 JSON 结果
        try {
            // 清理可能的 Markdown 代码块标记
            String json = result.replaceAll("```json\\s*", "").replaceAll("```\\s*", "");
            ProductRecognizeResult parsed = JsonUtil.parse(json, ProductRecognizeResult.class);

            // 根据识别结果匹配商品库
            if (parsed.keywords() != null && !parsed.keywords().isEmpty()) {
                List<Product> matched = productService.searchByKeywords(parsed.keywords());
                parsed = parsed.withMatchedProducts(matched);
            }

            return parsed;
        } catch (Exception e) {
            log.warn("【商品识别】结果解析失败: {}", e.getMessage());
            return new ProductRecognizeResult("未知", "未知", null, null, result,
                    List.of(), List.of());
        }
    }

    /**
     * 图片预处理：压缩到合理大小
     * 模型一般支持 10MB 以内的图片，但太大的图会增加处理时间和 Token 消耗
     */
    private byte[] preprocessImage(MultipartFile file) {
        try {
            BufferedImage original = ImageIO.read(file.getInputStream());

            // 如果宽度超过 1024px，等比缩放
            int maxWidth = 1024;
            if (original.getWidth() > maxWidth) {
                int newHeight = (int) ((double) original.getHeight() / original.getWidth() * maxWidth);
                BufferedImage resized = new BufferedImage(maxWidth, newHeight, original.getType());
                Graphics2D g = resized.createGraphics();
                g.setRenderingHint(RenderingHints.KEY_INTERPOLATION,
                        RenderingHints.VALUE_INTERPOLATION_BILINEAR);
                g.drawImage(original, 0, 0, maxWidth, newHeight, null);
                g.dispose();

                ByteArrayOutputStream baos = new ByteArrayOutputStream();
                ImageIO.write(resized, "jpg", baos);
                byte[] result = baos.toByteArray();
                log.info("【图片预处理】压缩: {}KB -> {}KB",
                        file.getSize() / 1024, result.length / 1024);
                return result;
            }

            return file.getBytes();
        } catch (IOException e) {
            log.warn("【图片预处理】压缩失败，使用原图: {}", e.getMessage());
            try {
                return file.getBytes();
            } catch (IOException ex) {
                throw new RuntimeException("读取图片失败", ex);
            }
        }
    }

    public record ProductRecognizeResult(
            String category,
            String subcategory,
            String brand,
            String color,
            String description,
            List<String> keywords,
            List<Product> matchedProducts
    ) {
        public ProductRecognizeResult withMatchedProducts(List<Product> products) {
            return new ProductRecognizeResult(category, subcategory, brand, color,
                    description, keywords, products);
        }
    }
}
```

### 第三步：OCR 增强

```java
@Service
@Slf4j
@RequiredArgsConstructor
public class SmartOcrService {

    private final ChatModel chatModel;

    /**
     * 快递单号识别
     * 用户拍照上传快递面单，自动提取单号
     */
    public ExpressInfo extractExpressNo(MultipartFile image) {
        String result = analyzeImage(image, """
                请识别这张快递面单图片，提取以下信息（JSON格式）：
                {
                    "expressNo": "快递单号",
                    "company": "快递公司名称",
                    "senderCity": "寄件城市",
                    "receiverCity": "收件城市",
                    "receiverName": "收件人（如有）"
                }
                如果某个字段无法识别，填 null。
                """);

        try {
            String json = result.replaceAll("```json\\s*", "").replaceAll("```\\s*", "");
            return JsonUtil.parse(json, ExpressInfo.class);
        } catch (Exception e) {
            return new ExpressInfo(null, null, null, null, null);
        }
    }

    /**
     * 通用 OCR：提取图片中的所有文字
     * 比传统 OCR 的优势是能理解表格、手写等复杂排版
     */
    public String extractText(MultipartFile image, String instruction) {
        String prompt = StringUtils.hasText(instruction)
                ? instruction
                : "请识别并提取图片中的所有文字内容，保持原始排版格式。";
        return analyzeImage(image, prompt);
    }

    /**
     * 表格图片转结构化数据
     */
    public String extractTable(MultipartFile image) {
        return analyzeImage(image, """
                请识别图片中的表格，转为 JSON 数组格式。
                每行是一个对象，key 是列名，value 是单元格内容。
                保持原始数据类型（数字用数字，文字用字符串）。
                """);
    }

    private String analyzeImage(MultipartFile image, String prompt) {
        try {
            byte[] bytes = image.getBytes();
            String mediaType = image.getContentType() != null
                    ? image.getContentType() : "image/jpeg";

            UserMessage message = UserMessage.builder()
                    .text(prompt)
                    .media(new Media(MimeTypeUtils.parseMimeType(mediaType),
                            new ByteArrayResource(bytes)))
                    .build();

            return chatModel.call(new Prompt(message))
                    .getResult().getOutput().getContent();
        } catch (IOException e) {
            throw new RuntimeException("读取图片失败", e);
        }
    }

    public record ExpressInfo(String expressNo, String company,
                               String senderCity, String receiverCity, String receiverName) {}
}
```

### 第四步：对外接口

```java
@RestController
@RequestMapping("/api/image")
@RequiredArgsConstructor
@Slf4j
public class ImageController {

    private final ProductRecognitionService productRecognitionService;
    private final SmartOcrService smartOcrService;

    /**
     * 商品识别接口
     */
    @PostMapping("/recognize-product")
    public ProductRecognitionService.ProductRecognizeResult recognizeProduct(
            @RequestParam("image") MultipartFile image) {
        return productRecognitionService.recognizeProduct(image);
    }

    /**
     * 快递单识别接口
     */
    @PostMapping("/extract-express")
    public SmartOcrService.ExpressInfo extractExpress(
            @RequestParam("image") MultipartFile image) {
        return smartOcrService.extractExpressNo(image);
    }

    /**
     * 通用图片文字提取
     */
    @PostMapping("/extract-text")
    public Map<String, String> extractText(
            @RequestParam("image") MultipartFile image,
            @RequestParam(value = "instruction", required = false) String instruction) {
        String text = smartOcrService.extractText(image, instruction);
        return Map.of("text", text);
    }
}
```

## 踩坑总结

1. **通义千问 VL 的图片尺寸限制**：DashScope 对图片大小有限制，单张图片不超过 10MB。但实际测试发现，超过 5MB 后响应时间明显变长。建议在后端做预处理，压缩到 1MB 以内。

2. **图片分辨率与 Token 消耗**：多模态模型会把图片转成 Token，分辨率越高 Token 越多。一张 1024x1024 的图大约消耗 700-1000 个 Token。如果不需要高清识别，压缩到 512x512 足够。

3. **JSON 输出不稳定**：让模型输出 JSON 格式的结果，有时候它会加 Markdown 代码块（```json ... ```），有时候不会。后端必须做容错解析，去掉代码块标记、处理尾逗号等。

4. **图片上传要限流**：图片识别的 Token 消耗是纯文字的 5-10 倍，成本很高。必须做限流，每个用户每天最多上传 N 张图片。另外要防恶意上传超大文件。

5. **Prompt 要具体**：识别商品图片时，Prompt 写"识别这张图片"效果很差。必须明确告诉模型你要什么信息、输出什么格式。我们的经验是 Prompt 越具体，输出的结构化程度越高。

6. **多模态模型的延迟较高**：图片理解通常比纯文字慢 2-3 倍。要做好超时处理和用户等待提示。我们的方案是先快速返回"正在识别图片..."的即时反馈，后台异步处理完成后通过 WebSocket 推送结果。

7. **Base64 传输的内存问题**：大图片 Base64 编码后体积膨胀约 33%，直接放 JSON 里传输会导致内存压力。建议用 Multipart 文件上传，不要用 Base64 JSON。
